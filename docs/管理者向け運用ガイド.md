# NICOX 管理者向け運用ガイド

本ドキュメントは、NICOX（地域猫マネジメントシステム）の **管理者** が日常業務で行う操作をまとめたものです。
管理パネル（`/admin`）にログインできる「管理者」ロールのユーザーを対象としています。

## 1. はじめに

### システムの概要

NICOX は、市民・活動者・行政が連携して野良猫問題に取り組むためのプラットフォームです。

- **市民** … 野良猫の目撃情報を「通報」として投稿
- **活動者** … 通報内容を地図で確認し、TNR活動や捕獲を実施
- **管理者** … システム全体を運用し、ユーザー承認・エリア定義・データ管理を行う

### 3層の情報公開レベル

| レベル | 閲覧者 | 見える情報 |
|---|---|---|
| **Level 1** | 一般公開（未ログイン） | 集計値のみ（通報総数、活動エリア数 など） |
| **Level 2** | 承認済みの活動者 | 通報の地図上の位置、写真、詳細 |
| **Level 3** | 管理者 | すべての情報＋ユーザー管理・監査ログ |

動物愛護の観点から、位置情報を含む詳細データは承認制で運用しています。

## 2. ログイン・ログアウト

### ログイン
1. トップページ右上の「ログイン / 新規登録」をクリック
2. 管理者として登録済みのメールアドレスとパスワードを入力
3. 「ログイン」を押下

### ログアウト
1. 画面右上のメニューから「ログアウト」をクリック
2. 確認モーダルで「ログアウト」を選択
3. 完了通知（トースト）が画面右上に表示されます

> **注意**：ログアウトすると、再度ログインするまで Level 2 以上の情報は閲覧できません。

---

## 3. 管理パネルの構成

ログイン後、画面右上の「管理パネル」リンクから `/admin` にアクセスできます。
左側のタブで以下のセクションを切り替えます。

| タブ | 用途 |
|---|---|
| **ダッシュボード** | 全体の集計値、直近の通報を一覧 |
| **ユーザー** | アカウントの承認・ロール変更・停止 |
| **通報管理** | 通報の検索・確認・ステータス更新・削除 |
| **ホットスポット** | 通報の集積エリアを自動生成・管理 |
| **エリア** | 行政区など、地理的なエリア（ポリゴン）を定義 |
| **監査ログ** | 管理者操作の履歴を確認 |
| **エクスポート** | 通報データを CSV で出力 |

## 4. ダッシュボード

### 表示内容
- **集計カード**：通報総数、未対応の通報、活動中のエリア、要対応エリアなど
- **直近の通報（最新10件）**：通報ID、通報者、時刻、エリア、座標、問題種別、写真枚数、ステータス

### よく使う操作
- **通報行をクリック** … 通報詳細モーダルが開き、写真・詳細情報・管理メモが確認できます
- **更新ボタン** … 集計値・直近通報を再取得

> ステータスは色付きの「ピル（バッジ）」で表示されます：
> - **未対応**（オレンジ）
> - **対応済み**（緑）
> - **破棄**（グレー）

## 5. ユーザー管理

### 5.1 新規登録ユーザーの承認

新規登録者は **承認待ち** 状態でシステムに入ります。承認するまで Level 2 以上の機能は使えません。

1. 「ユーザー」タブを開く
2. ステータス絞り込みで「**承認待ち**」を選択
3. 該当ユーザー行で **承認** または **却下** をクリック
4. 承認後、活動者ロールが必要であれば **役割変更** から「活動者」を選択

> **目安**：承認には数日〜1週間程度の余裕を持って対応してください。

### 5.2 ロール変更
- **市民**：通報のみ可能（デフォルト）
- **活動者**：通報＋通報詳細閲覧＋（割当エリアの）TNR等の活動
- **管理者**：すべての権限

ロール変更は「役割」のセレクトボックスから即時反映されます。

### 5.3 エリア割当
活動者は **割当エリア** 内の通報のみ詳細を見られます。

1. ユーザー行の **エリア割当** ボタンを開く
2. 担当するエリアをチェック → 保存

### 5.4 アカウント停止 / 再有効化
不正利用が疑われる場合は **停止** ボタンでアカウントを無効化できます。
停止中ユーザーはログインできません。再開する場合は同じボタンで再有効化します。

### 5.5 ユーザー削除
**削除** ボタンで完全に削除されます。投稿された通報は残ります（通報者名は「匿名」表示）。

## 6. 通報管理

### 6.1 通報の検索
- **検索ボックス**：通報者名・通報内容を部分一致で検索
- **期間**：通報日時で絞り込み
- **ステータス**：未対応 / 対応済み / 破棄
- **写真あり**：写真添付された通報のみ
- **エリア**：特定エリア内の通報のみ

### 6.2 通報詳細の確認
通報行右の **目アイコン** をクリックすると、詳細モーダルが開きます。

表示内容：
- 通報者情報（名前・メール・電話）
- 通報日時、ステータス（ピル表示）
- 該当エリア
- 座標（NSEW形式：例 `35.7045°N, 139.6947°E`）
- 頭数、耳カット状況、子猫の有無
- 問題種別（糞尿被害、鳴き声被害 など）
- 要望（被害を減らしたい、手術をしたい など）
- 写真（クリックで拡大）
- 管理メモ（自由記述・保存可能）

### 6.3 ステータス変更・削除
- ステータスは詳細モーダル内で **未対応 → 対応済み / 破棄** に変更可能
- 一覧の **ゴミ箱アイコン** で個別削除
- チェックボックスで複数選択 →「一括削除」も可能

> **削除は取り消せません**。誤操作防止のため、確認ダイアログが表示されます。

## 7. ホットスポット管理

### 7.1 ホットスポットとは
通報が密集している地点を自動的にクラスタリング（集約）したものです。
活動者が「この地区を重点的に対応すべき」と判断するための指標になります。

### 7.2 再生成（リフレッシュ）
通報データから新しいホットスポットを生成します。

1. 「ホットスポット」タブを開く
2. パラメータを設定：
   - **遡及日数**：何日前までの通報を対象にするか（既定: 30日）
   - **クラスタ半径(m)**：何m以内の通報を1つにまとめるか（既定: 100m）
   - **最小通報数**：クラスタとして扱う最小通報数（既定: 1）
   - **既存を削除して再生成**：チェックすると古いホットスポットを全削除してから再構築
3. **再生成実行** をクリック

> **運用の目安**：週1回程度、または通報が一定数溜まったタイミングで実行してください。
> 「既存を削除」を有効にすると、ステータスや手動メモも消えるため、通常はオフのままを推奨。

### 7.3 ホットスポットのステータス
| ステータス | 意味 |
|---|---|
| **活動中** | 通常の対応中エリア |
| **監視中** | 様子見 |
| **高優先** | 緊急対応が必要 |
| **解決済み** | 対応完了 |

### 7.4 削除
個別のホットスポットを削除する場合は **ゴミ箱アイコン** から。

## 8. エリア管理

### 8.1 エリアとは
行政区・地区などの **ポリゴン（多角形）** をシステムに登録したものです。
通報・ホットスポットがどのエリアに属するかは、座標とポリゴンの包含関係（PostGIS の `ST_Contains`）で自動判定されます。

### 8.2 新規エリアの作成

1. 「エリア」タブを開く
2. **新規作成** ボタンをクリック
3. **エリア名**（例：渋谷区、中央地区）を入力
4. **エリア種別** を選択：地区 / 区 / 市 / カスタム
5. 地図上をクリックして **ポリゴンの頂点を順番に追加**
6. 最後の頂点をクリックすると自動で閉じます
7. **作成** ボタン（横幅100px）で保存

> **重要**：ポリゴンが小さすぎると、その範囲外の通報は含まれません。
> 通報が含まれるよう、対象地域を **十分に包含する大きさ** で描画してください。

### 8.3 エリアの編集・削除
- エリア行で **編集** … 名前・種別の変更
- **中心座標リンク**（例：`中心座標: 35.7045°N, 139.6947°E`）をクリックすると、ミニ地図でポリゴンの位置を確認できます
- **削除** … エリアを削除（通報・ホットスポットは残ります）

### 8.4 動作確認
エリア作成後、ダッシュボードや通報管理で「該当エリアの通報数」が反映されていることを確認してください。
0件のままなら、ポリゴンが通報の位置を **物理的に含んでいない** 可能性が高いです。

## 9. 監査ログ

### 表示内容
すべての管理者操作の履歴を記録しています。

| 列 | 内容 |
|---|---|
| 日時 | 操作日時 |
| 操作者 | 実行した管理者 |
| アクション | 何を行ったか（例：ユーザー承認、通報削除） |
| 対象 | 操作対象のID / 名前 |
| 詳細 | 変更内容（日本語ラベルに翻訳済み） |

### 用途
- 不正操作の調査
- 設定変更のトレース
- 監査対応のエビデンス

> 監査ログは自動的に記録されるため、管理者の操作内容は **すべて追跡可能** です。

## 10. 通報CSVエクスポート

### 出力方法
1. 「エクスポート」タブを開く
2. 期間・ステータスなどの絞り込みを指定
3. **エクスポート実行** をクリック → CSV ファイルがダウンロードされます

### CSV の利用
- 行政への報告資料作成
- Excel での集計・分析
- 外部システムへの連携

> **個人情報を含む** ため、ダウンロードしたCSVは厳重に管理してください。

## 11. よくある質問・トラブルシューティング

### Q1. 新規登録の確認メールが届かない
- **迷惑メールフォルダ** を確認してください
- メールアドレスに誤字がないか確認
- それでも届かない場合は管理者に連絡し、Supabase のメール送信設定を確認してもらってください

### Q2. 「メール送信回数の上限を超えました」と出る
Supabase のメール送信レート制限です。
- 1時間ほど待ってから再度お試しください
- 頻発する場合は管理者がカスタムSMTP（SendGrid 等）の設定を行う必要があります

### Q3. 「セキュリティ保護のため、◯秒後に再度お試しください」と出る
同じ操作の連続実行に対する保護です。表示されている秒数だけ待ってから再試行してください。

### Q4. ダッシュボードの「エリア別」値がすべて 0 になっている
通報座標が **どのエリアのポリゴン内にも入っていない** 場合に発生します。
- 「エリア管理」で各エリアのポリゴンを確認
- 必要に応じてポリゴンを再描画、または新規通報を該当エリア内で投稿

### Q5. 古い通報の問題種別が英語表示になっている
旧バージョンで登録された通報には英語コード（`waste`, `kittens` など）が含まれます。
表示マップに対応コードを追加済みのため、最新の管理パネルでは日本語表示されます。
それでも英語表示が残る場合は、ブラウザで **ハードリロード**（`Ctrl+Shift+R`）してください。

### Q6. ホットスポットが想定通りに生成されない
- 「最小通報数」が高すぎる → 1〜2 に下げる
- 「クラスタ半径」が小さすぎる → 100〜500m を試す
- 「遡及日数」内に通報がない → 期間を延ばす

### Q7. ユーザーを誤って削除してしまった
削除は **取り消せません**。再登録してもらう必要があります。
通報データは残っているため、紐付けし直すには手動でのDB操作が必要です（管理者に依頼）。
